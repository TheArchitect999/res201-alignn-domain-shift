# Grand Plan v2: A→Z pipeline for two high-grade RES201 reports

## 1. Non-negotiable operating rules

1. **Evidence first, prose second.**
   No section gets drafted before the source files, canonical numbers, and main figures are frozen.

2. **One primary repo agent per stage.**
   Use **Codex** as the main repo operator. Use **Claude Code** as the auditor/refiner after Codex, not in parallel on the same raw stage.

3. **Every major figure gets a Figure Memo before it enters the paper.**
   This is the main upgrade to the previous plan.

4. **Every section must separate three layers clearly:**

   * literature-grounded claims
   * our implementation details
   * our experimental findings

5. **Use the project-brief hyperparameter setting as the main narrative basis.**
   The project brief fixes the main experimental condition at **epochs = 50, lr = 1e-4, batch size = 16**, so the repo namespace matching that setting should become the canonical main-results namespace during Phase 1. 

6. **Keep oxide and nitride aligned but not cloned.**
   Shared methods backbone: yes.
   Same full paragraphs and same report emphasis: no.

7. **Treat embedding plots carefully.**
   Week 2 already warns that t-SNE and UMAP are useful but must be interpreted cautiously; t-SNE especially preserves local neighborhoods but not global inter-cluster distances in a physically meaningful way. 

8. **Move into the JURI template late.**
   The JURI template is the formatting shell, not the drafting environment. The abstract must be a single paragraph and should stay under 250 words.

By the end, you should have **three complete products**:

1. **Oxide standalone report**
2. **Nitride standalone report**
3. **Combined joint paper draft**

The project brief explicitly wants:

* written oxide analysis
* written nitride analysis
* oxide results draft
* nitride results draft
* and a full combined paper with **Introduction, Methods, Results I oxide, Results II nitride, Results III direct comparison, Results IV embedding analysis, Discussion, Conclusion, References**. 

The JURI template then determines the final presentation format.

# 7. Exactly how to execute this in practice

5. **Send the evidence pack to Claude Pro** only after the evidence is frozen.
6. **Use Prompt T4** to let Codex or Claude Code apply approved fixes.
7. **Move to the next phase only after the stop/go checkpoint is satisfied.**

For image-heavy phases:

1. Codex identifies the figure path and supporting table
2. You upload figure + table to ChatGPT with **Prompt T1**
3. You upload figure + table to Claude Pro with **Prompt T1**
4. Compare outputs
5. Codex consolidates the memo

That is the cleanest premium pipeline that uses vision without making the workflow chaotic.
