## Our Implementation Details

### Dataset source and target

- Both family datasets were derived from `dft_3d_2021`.
- The prediction target was `formation_energy_peratom`.
- Family-level summaries are recorded in:
  - `data_shared/oxide/summaries/summary.json`
  - `data_shared/nitride/summaries/summary.json`

### Family definitions and filtering

- Oxide arm: structures containing `O`.
- Nitride arm: structures containing `N` and not containing `O`.
- Oxynitrides remain in the oxide arm when oxygen is present and are excluded from the nitride arm.
- The oxide summary records `499` oxynitrides; the nitride summary records `0`.

### Split protocol

- Original split identities were preserved before family filtering.
- The recorded split source is `provided:manifests/dft_3d_formation_energy_peratom_splits.csv`.
- For each family:
  - `pool = train + val`
  - `test` remains fixed
- Run-local dataset roots are generated by sampling `N` structures from the family pool with the run seed, assigning roughly `10%` to validation with a minimum of `5` validation examples, and appending the unchanged family test set.
- `keep_data_order=True` plus explicit `n_train`, `n_val`, and `n_test` are used to preserve the train/val/test order inside `id_prop.csv`.

### Zero-shot baseline

- The zero-shot baseline uses the fixed family test manifests and `alignn.pretrained.get_prediction(...)`.
- MAE is computed per family and recorded in `reports/zero_shot/zero_shot_summary.csv`.
- Per-structure zero-shot outputs live in `Results_Before_Correction/<family>/zero_shot/predictions.csv`.

### Fine-tuning protocol

- Fine-tuning loads the pretrained checkpoint `jv_formation_energy_peratom_alignn/checkpoint_300.pt` and its config.
- All parameters are frozen before selectively unfreezing:
  - `gcn_layers.3`
  - `fc`
- The custom script keeps the overall model in `eval()` mode, then re-enables training only for the unfrozen modules.
- Optimization uses `AdamW`, `OneCycleLR`, and `L1Loss`.
- Best-checkpoint selection is based on the lowest validation `L1`.

### From-scratch protocol

- From-scratch training reuses the same run-local dataset roots and model config but initializes a fresh ALIGNN model with no pretrained weights.
- All parameters are trainable.
- Optimization uses the same `AdamW` plus `OneCycleLR` schedule and `L1Loss` objective as the partial fine-tuning script.
- Best-checkpoint selection is again based on validation `L1`.

### Hyperparameter setting used for the main narrative

- The combined paper should explicitly name Hyperparameter Set 1 as the canonical namespace:
  - `epochs = 50`
  - `batch_size = 16`
  - `learning_rate = 1e-4`
- Shared config settings visible in representative Set 1 configs:
  - `neighbor_strategy = "k-nearest"`
  - `optimizer = "adamw"`
  - `scheduler = "onecycle"`
  - `cutoff = 8.0`
  - `cutoff_extra = 3.0`
  - `max_neighbors = 12`
  - `use_canonize = true`
  - `compute_line_graph = true`
  - `use_lmdb = true`
- Shared model specification:
  - four ALIGNN layers
  - four GCN layers
  - hidden size `256`
  - scalar output

### Evaluation metric

- The primary reported metric is test MAE in `eV/atom`.
- Fine-tuning and from-scratch checkpoint selection use validation `L1`.

### Embedding-analysis protocol

- Embeddings are extracted from the frozen pretrained ALIGNN model, not from fine-tuned or from-scratch checkpoints.
- Model source:
  - checkpoint: `jv_formation_energy_peratom_alignn/checkpoint_300.pt`
  - config: `jv_formation_energy_peratom_alignn/config.json`
- Embedding sources (all three extracted; `last_alignn_pool` is the primary main-text layer):
  - `pre_head`: pooled output of `model.readout` immediately before `model.fc`
  - `last_alignn_pool`: pooled node tensor returned by the last `ALIGNNConv` block (`alignn_layers[3]`), before the gated GCN stack
  - `last_gcn_pool`: pooled node tensor returned by the last gated GCN block (`gcn_layers[3]`); in this model it is a near-duplicate of `pre_head`
- Named subsets:
  - `fixed_test_set`
  - `balanced_pool_set`
  - `oxide_reference_pool`
- Quantitative family-separation metrics on raw 256D embeddings:
  - silhouette score
  - Davies-Bouldin index
  - `k = 15` nearest-neighbor family purity
  - logistic-regression AUC with `5`-fold cross-validation and `StandardScaler`
- Quantitative nitride distance-versus-error metrics:
  - oxide centroid Euclidean distance
  - mean Euclidean distance to the `5` nearest oxide-reference embeddings
  - supplemental Ledoit-Wolf Mahalanobis distance if covariance screening passes
- Visualization workflow:
  - PCA: standardize first, fit PCA on `balanced_pool_set`, project other subsets into that basis
  - t-SNE: standardize first, no PCA pre-reduction, main perplexity `30`, sensitivity `15` and `50`
  - UMAP: standardize first, no PCA pre-reduction, main `n_neighbors = 30`, `min_dist = 0.1`, sensitivity `n_neighbors = 15` and `50`
- Overlay policy:
  - t-SNE hard/easy overlay is refit directly on `balanced_pool_set + fixed-test nitride`
  - UMAP hard/easy overlay is fit on standardized `balanced_pool_set`, then fixed-test nitrides are transformed onto that manifold
- Statistical defaults:
  - family-separation bootstraps: `1000`
  - distance-error bootstraps: `5000`
  - distance-error permutations: `10 000`
  - within-statistic Benjamini-Hochberg FDR adjustment
- Hard/easy nitride groups use the top and bottom `20%` of fixed-test nitride absolute zero-shot error.
- Raw-space statistics are inferential; 2D projections are descriptive support.
